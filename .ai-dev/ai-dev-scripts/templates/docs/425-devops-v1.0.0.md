## CI/CD Pipeline

### Pull Request Pipeline

```
push to branch
  → lint + type check
  → unit tests
  → integration tests
  → build (verify no compile errors)
  → coverage check (fail if < 80%)
```

### Merge to Main Pipeline

```
merge to main
  → full test suite (unit + integration + E2E)
  → build Docker image
  → push to registry
  → deploy to staging (auto)
  → smoke tests on staging
```

### Production Release Pipeline

```
tag vX.Y.Z
  → full test suite
  → build + tag Docker image with version
  → deploy to production (manual approval gate)
  → post-deploy health check
  → notify team
```

## Docker

```dockerfile
# Example structure
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM node:20-alpine AS runner
WORKDIR /app
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/node_modules ./node_modules
CMD ["node", "dist/main.js"]
```

## PR Workflow

1. Branch from `main` using `feature/`, `fix/`, `chore/` prefix
2. Open PR — CI pipeline runs automatically
3. Requires 1 approval + passing CI to merge
4. Squash merge into `main`
5. Branch auto-deleted after merge

## Rollback Procedures

1. Identify bad deploy from monitoring
2. Trigger rollback: redeploy previous Docker image tag
3. Verify health checks pass
4. Investigate root cause before re-deploying fix

## Release Strategy

- Semantic versioning: `vMAJOR.MINOR.PATCH`
- Release notes generated by `/release`
- Git tag created on every production release
- Docker image tagged with version + `latest`
